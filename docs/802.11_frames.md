# Référence — Trames 802.11

## Structure MAC Header (24 bytes)

```
Offset  Taille  Champ
──────────────────────────────────────────────────
0       2       Frame Control (FC)
2       2       Duration / ID
4       6       Adresse 1 (destination)
10      6       Adresse 2 (source / émetteur)
16      6       Adresse 3 (BSSID ou destination)
22      2       Sequence Control
24+             Corps de trame (frame body)
```

## Frame Control — byte 0

```
Bits 0-1 : Protocol version  (toujours 00)
Bits 2-3 : Type
           00 = Management (MGMT)
           01 = Control    (CTRL)
           10 = Data       (DATA)
Bits 4-7 : Subtype (voir tableaux ci-dessous)

Valeur byte 0 = (subtype << 4) | (type << 2) | version
```

## Frame Control — byte 1

```
Bit 0 : To DS
Bit 1 : From DS
Bit 2 : More Fragments
Bit 3 : Retry
Bit 4 : Power Management
Bit 5 : More Data
Bit 6 : Protected (WEP/WPA/WPA2)
Bit 7 : Order
```

## Sous-types Management (type=00)

| Subtype | Hex FC[0] | Nom           | Usage lab                      |
|---------|-----------|---------------|--------------------------------|
| 0       | 0x00      | Assoc Req     | Station → AP pour s'associer   |
| 1       | 0x10      | Assoc Resp    | AP → Station (accepté/refusé)  |
| 4       | 0x40      | Probe Req     | Station cherche un réseau      |
| 5       | 0x50      | Probe Resp    | AP répond à un Probe Req       |
| 8       | 0x80      | Beacon        | AP annonce sa présence         |
| 10      | 0xA0      | Disassoc      | Désassociation                 |
| 11      | 0xB0      | Auth          | Authentification               |
| 12      | 0xC0      | **Deauth**    | **Déauthentification → attaque**|

## Sous-types Data (type=10)

| Subtype | Hex FC[0] | Nom      | Contient EAPOL ? |
|---------|-----------|----------|-----------------|
| 0       | 0x08      | Data     | Possible (WPA2 handshake) |
| 8       | 0x88      | QoS Data | Possible         |

## Trame Deauthentication

```
FC[0]    = 0xC0  (MGMT, subtype=12)
FC[1]    = 0x00
Duration = 0x0000
Addr1    = MAC cible (ou FF:FF:FF:FF:FF:FF broadcast)
Addr2    = MAC AP (spoofé)
Addr3    = BSSID (= MAC AP)
SeqCtrl  = 0x0000+
Body     = Reason Code (2 bytes LE)
```

### Reason Codes courants

| Code | Signification                                |
|------|----------------------------------------------|
| 1    | Unspecified                                  |
| 2    | Auth expired                                 |
| 3    | Leaving BSS (désassociation propre)          |
| 7    | **Class 3 frame from nonassociated STA** ← utilisé dans deauth.py |
| 8    | Disassociated: leaving BSS                   |

## Trame Beacon

```
FC[0]    = 0x80  (MGMT, subtype=8)
FC[1]    = 0x00
Duration = 0x0000
Addr1    = FF:FF:FF:FF:FF:FF  (broadcast)
Addr2    = BSSID
Addr3    = BSSID
SeqCtrl  = variable
─── Corps ───
Timestamp       8 bytes   (µs depuis démarrage AP)
Beacon Interval 2 bytes   (100 TU = 0x0064 ≈ 102.4ms)
Capability Info 2 bytes   (0x0401=ESS open, 0x0431=ESS+Privacy)
IE SSID         type=0, len, SSID
IE Rates        type=1, len=8, taux supportés
IE DS Param     type=3, len=1, canal
```

## EAPOL — Handshake WPA2 (4-way)

Détecté dans les trames Data via le marqueur LLC/SNAP :
```
AA AA 03 00 00 00 88 8E
```

```
Message 1 : AP  → STA  (ANonce)
Message 2 : STA → AP   (SNonce + MIC)   ← contient le hash crackable
Message 3 : AP  → STA  (GTK chiffré)
Message 4 : STA → AP   (ACK)
```

Avec 2 messages (1+2 ou 2+3), on peut tenter le crack PMKID/EAPOL.

## Adresses dans une trame Data

| ToDS | FromDS | Addr1  | Addr2  | Addr3  | Addr4  |
|------|--------|--------|--------|--------|--------|
| 0    | 0      | DA     | SA     | BSSID  | —      |
| 0    | 1      | DA     | BSSID  | SA     | —      |
| 1    | 0      | BSSID  | SA     | DA     | —      |
| 1    | 1      | RA     | TA     | DA     | SA     |
